# 删除过时菜单说明

## 背景

随着用户管理功能的完善，以下三个独立的列表页面已经不再需要：

1. **禁言列表** (`/moderation/mute-list`)
2. **封禁列表** (`/moderation/ban-list`)  
3. **AI自动禁言** (`/moderation/auto-mute-list`)

这些功能已经整合到"违规处理"页面中，通过统一的用户管理界面可以完成所有禁言和封禁操作。

## 删除内容

### 1. 菜单初始化配置

**文件：** `backend/mainotebook/system/fixtures/init_menu.json`

删除了三个菜单及其按钮配置：
- 禁言列表菜单（包含6个按钮）
- 封禁列表菜单（包含6个按钮）
- AI自动禁言菜单（包含5个按钮）

### 2. 角色菜单权限配置

**文件：** `backend/mainotebook/system/fixtures/init_rolemenupermission.json`

删除了以下角色对这三个菜单的权限：
- `superadmin` 角色
- `admin` 角色
- `moderation_admin` 角色

### 3. 数据库清理

**脚本：** `backend/remove_obsolete_menus.py`

创建了清理脚本，用于删除现有数据库中的：
- 3个菜单记录
- 17个菜单按钮记录
- 9条角色菜单权限记录
- 51条角色按钮权限记录

## 保留内容

### API 端点

以下 API 端点暂时保留，以防前端或其他服务仍在使用：

- `GET /api/content/moderation/mute-list/` - 禁言列表查询
- `GET /api/content/moderation/ban-list/` - 封禁列表查询
- `GET /api/content/moderation/auto-mute-list/` - AI自动禁言列表查询

这些端点在 `UserModerationViewSet` 中实现，如果确认不再使用，可以在后续版本中删除。

### 角色

`moderation_admin` 角色保留，该角色仍然拥有"违规处理"父菜单的访问权限。

## 执行步骤

### 对于新部署

新部署会自动使用更新后的初始化配置，不会创建这三个菜单。

### 对于现有部署

需要手动运行清理脚本：

```bash
conda activate mai_notebook
cd backend
python remove_obsolete_menus.py
```

## 影响范围

### 前端

需要删除或隐藏以下前端页面：
- `admin_web/src/views/content/moderation/mute/index.vue`
- `admin_web/src/views/content/moderation/ban/index.vue`
- `admin_web/src/views/content/moderation/auto-mute/index.vue`

### 用户体验

用户现在通过"违规处理"页面统一管理所有禁言和封禁操作，界面更加简洁统一。

## 后续清理

如果确认 API 端点不再使用，可以在后续版本中：

1. 删除 `UserModerationViewSet` 中的以下方法：
   - `mute_list()`
   - `ban_list()`
   - `auto_mute_list()`

2. 更新 API 文档，移除这些端点的说明
