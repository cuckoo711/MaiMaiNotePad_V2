## 📝 文档维护规范

### 文档分类原则

1. **功能文档** (`功能文档/`)
   - 面向用户和开发者的功能说明
   - 持续维护和更新的文档
   - 包含使用指南、API 说明等

2. **技术文档** (`技术文档/`)
   - 架构设计、技术实现细节
   - 配置说明、部署指南
   - 技术决策和优化方案

3. **归档文档** (`archive/`)
   - 历史性、任务型文档
   - 已完成的功能开发记录
   - 测试报告、迁移记录
   - 不再频繁更新的文档

### 文档编写规范

- 所有文档使用简体中文
- 文档文件名使用中文
- 创建或删除文档时同步更新本索引
- 确保文档包含清晰的标题层级
- 临时性文档完成后归档到 `archive/` 目录
