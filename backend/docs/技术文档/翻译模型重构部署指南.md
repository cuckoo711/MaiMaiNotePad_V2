# 翻译模型重构部署指南

## 概述

本文档提供翻译模型重构功能的完整部署指南，包括数据库迁移、数据迁移、API 配置、前端部署等步骤。

### 功能说明

翻译模型重构将翻译功能从字典模型（Dictionary）中解耦，创建独立的翻译模型（Translation）来管理所有翻译内容。主要变更包括：

- 创建独立的 Translation 数据模型
- 实现翻译数据的 CRUD API
- 从 Dictionary 模型迁移翻译数据
- 更新前端代码使用新的翻译 API
- 配置 API 白名单支持公开访问

### 影响范围

- **后端**: 新增 Translation 模型、API 接口、数据迁移脚本
- **前端**: 更新 TOML Visualizer 组件使用新的翻译 API
- **数据库**: 新增 Translation 表，迁移字典表中的翻译数据

## 回滚方案

如果部署后发现问题，可以按以下步骤回滚：

### 方案一：保留 Translation 表，恢复字典数据

如果只是数据问题，可以从备份恢复字典数据：

```bash
# 从备份文件恢复字典数据
python manage.py shell

>>> import json
>>> from mainotebook.system.models import Dictionary
>>> 
>>> # 读取备份文件
>>> with open('translation_backup_YYYYMMDD_HHMMSS.json', 'r', encoding='utf-8') as f:
...     backup_data = json.load(f)
>>> 
>>> # 恢复数据
>>> for item in backup_data:
...     parent = None
...     if item['parent_value']:
...         parent = Dictionary.objects.filter(value=item['parent_value']).first()
...     Dictionary.objects.create(
...         label=item['label'],
...         value=item['value'],
...         parent=parent,
...         type=item['type'],
...         sort=item['sort'],
...         status=item['status']
...     )
>>> 
>>> exit()
```

### 方案二：完全回滚

如果需要完全回滚到旧版本：

```bash
# 1. 回滚代码
git revert <commit-hash>
git push

# 2. 回滚数据库迁移
python manage.py migrate system <previous_migration_number>

# 3. 恢复数据库备份（如果需要）
psql -U postgres -d mainotebook < backup_YYYYMMDD_HHMMSS.sql

# 4. 重启服务
bash start_backend.sh

# 5. 重新构建前端
cd admin_web
npm run build
```

## 故障排查

### 问题 1：数据迁移失败

**症状**: 执行 `migrate_translation` 命令时报错

**可能原因**:
- Dictionary 表中没有翻译数据
- 数据库连接问题
- 权限问题

**解决方法**:

```bash
# 1. 检查 Dictionary 表数据
python manage.py shell
>>> from mainotebook.system.models import Dictionary
>>> Dictionary.objects.filter(value='toml_visualizer_blocks').exists()
>>> Dictionary.objects.filter(value='toml_visualizer_tokens').exists()
>>> exit()

# 2. 检查数据库连接
python manage.py dbshell

# 3. 查看详细错误日志
python manage.py migrate_translation --dry-run
```

### 问题 2：API 返回 403 错误

**症状**: 访问 `/api/translations/` 返回 403 Forbidden

**可能原因**:
- API 白名单配置不正确
- 权限配置错误

**解决方法**:

```bash
# 检查白名单配置
python manage.py shell
>>> from mainotebook.system.models import ApiWhiteList
>>> ApiWhiteList.objects.filter(url__contains='translations').values()
>>> exit()

# 如果没有记录，添加白名单
# 参考"第四步：API 白名单配置"
```

### 问题 3：前端显示英文而非中文

**症状**: TOML 可视化解析器显示英文原文，没有中文翻译

**可能原因**:
- 翻译数据未加载
- API 请求失败
- 翻译 Store 未正确初始化

**解决方法**:

1. 打开浏览器开发者工具（F12）
2. 查看 Network 标签，检查 API 请求：
   - 是否发送了 `/api/translations/get_by_type/` 请求
   - 请求是否成功（状态码 200）
   - 响应数据是否正确
3. 查看 Console 标签，检查是否有 JavaScript 错误
4. 检查 Vue DevTools，查看 Translation Store 状态

### 问题 4：数据库查询慢

**症状**: API 响应时间过长

**可能原因**:
- 索引未创建
- 数据量过大
- 数据库性能问题

**解决方法**:

## 常见问题

### Q1: 迁移后可以删除 Dictionary 表吗？

**A**: 不可以。Dictionary 表还用于其他功能（如系统配置、数据字典等），只能删除翻译相关的记录。

### Q2: 如何添加新的翻译？

**A**: 有两种方式：

1. 通过 Django Admin 后台添加
2. 通过 API 添加（需要管理员权限）

```bash
# 方式 1：Django Admin
# 访问 http://localhost:8000/admin/system/translation/

# 方式 2：API
curl -X POST http://localhost:8000/api/translations/ \
  -H "Authorization: JWT <admin_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "source_text": "new_key",
    "translated_text": "新键",
    "translation_type": "toml_visualizer_tokens",
    "source_language": "en",
    "target_language": "zh",
    "sort": 100,
    "status": true
  }'
```

### Q3: 如何修改现有翻译？

**A**: 通过 Django Admin 后台或 API 修改（需要管理员权限）。

### Q4: 前端缓存了旧的翻译数据怎么办？

**A**: 清除浏览器缓存或使用隐私模式重新访问。

### Q5: 如何支持更多语言？

**A**: 修改 `source_language` 和 `target_language` 字段，添加新的语言代码（如 `ja` 日语、`ko` 韩语等）。

### C. 数据库表结构

```sql
CREATE TABLE dvadmin_system_translation (
    id UUID PRIMARY KEY,
    source_text VARCHAR(200) NOT NULL,
    translated_text VARCHAR(200) NOT NULL,
    source_language VARCHAR(10) NOT NULL DEFAULT 'en',
    target_language VARCHAR(10) NOT NULL DEFAULT 'zh',
    translation_type VARCHAR(50) NOT NULL,
    sort INTEGER NOT NULL DEFAULT 1,
    status BOOLEAN NOT NULL DEFAULT TRUE,
    creator_id UUID,
    modifier VARCHAR(255),
    dept_belong_id VARCHAR(255),
    create_datetime TIMESTAMP NOT NULL,
    update_datetime TIMESTAMP NOT NULL,
    description VARCHAR(255),
    UNIQUE (translation_type, source_text)
);

CREATE INDEX idx_translation_type ON dvadmin_system_translation(translation_type);
CREATE INDEX idx_source_language ON dvadmin_system_translation(source_language);
CREATE INDEX idx_target_language ON dvadmin_system_translation(target_language);
```
