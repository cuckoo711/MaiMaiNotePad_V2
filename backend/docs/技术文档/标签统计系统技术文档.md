# 标签统计系统技术文档

## 概述

标签统计系统为知识库和人设卡提供热门标签查询、标签搜索统计、标签使用统计等功能。系统支持按内容类型隔离标签，并使用 Redis 缓存提升查询性能。

## 系统架构

### 数据模型

#### TagStatistics 模型

```python
class TagStatistics(CoreModel):
    """标签统计模型"""
    
    TAG_TYPE_CHOICES = [
        ('knowledge', '知识库'),
        ('persona', '人设卡'),
    ]
    
    tag = models.CharField(max_length=50, verbose_name="标签名称")
    tag_type = models.CharField(
        max_length=20,
        choices=TAG_TYPE_CHOICES,
        default='knowledge',
        verbose_name="标签类型"
    )
    usage_count = models.IntegerField(default=0, verbose_name="使用次数")
    search_count = models.IntegerField(default=0, verbose_name="搜索次数")
    hot_score = models.FloatField(default=0.0, verbose_name="热度分数")
    last_used_at = models.DateTimeField(null=True, blank=True, verbose_name="最后使用时间")
    
    class Meta:
        unique_together = [['tag', 'tag_type']]
        ordering = ("-hot_score", "-usage_count")
```

**字段说明：**

- `tag`: 标签名称，最大长度 50 字符
- `tag_type`: 标签类型，'knowledge' 或 'persona'
- `usage_count`: 标签在内容中的使用次数
- `search_count`: 用户搜索该标签的次数
- `hot_score`: 热度分数，综合使用次数和搜索次数计算
- `last_used_at`: 标签最后一次被使用的时间

**约束和索引：**

- `unique_together=['tag', 'tag_type']`: 同一标签在不同类型中可以独立存在
- 索引：`tag`, `tag_type`, `hot_score`, `usage_count`, `search_count`, `last_used_at`
- 默认排序：按热度分数和使用次数降序

### 热度分数计算

热度分数综合考虑使用次数和搜索次数：

```python
def calculate_hot_score(self) -> float:
    """计算热度分数
    
    热度分数 = 使用次数 * 0.6 + 搜索次数 * 0.4
    """
    return self.usage_count * 0.6 + self.search_count * 0.4
```

**权重说明：**
- 使用次数权重 60%：反映标签在内容中的实际使用频率
- 搜索次数权重 40%：反映用户对该标签的关注度

## 服务层设计

### TagService 类

`TagService` 提供标签统计的核心业务逻辑。

#### 缓存策略

```python
class TagService:
    # 缓存键前缀
    POPULAR_TAGS_CACHE_KEY_PREFIX = "content:popular_tags"
    # 缓存过期时间（秒）
    CACHE_TIMEOUT = 3600  # 1小时
```

**缓存键格式：**
- 知识库标签：`content:popular_tags:knowledge`
- 人设卡标签：`content:popular_tags:persona`

#### 核心方法

##### 1. 获取热门标签

```python
@classmethod
def get_popular_tags(cls, limit: int = 20, tag_type: str = 'knowledge') -> List[Dict[str, any]]:
    """获取热门标签列表
    
    优先从缓存读取，缓存未命中时从数据库查询并更新缓存。
    
    Args:
        limit: 返回的标签数量，默认20个
        tag_type: 标签类型，'knowledge' 或 'persona'，默认 'knowledge'
        
    Returns:
        List[Dict]: 热门标签列表
    """
```

**查询流程：**
1. 尝试从 Redis 缓存获取
2. 缓存命中：直接返回结果
3. 缓存未命中：查询数据库
4. 更新缓存（过期时间 1 小时）
5. 返回结果

##### 2. 增加搜索次数

```python
@classmethod
def increment_search_count(cls, tag: str, tag_type: str = 'knowledge') -> None:
    """增加标签搜索次数
    
    当用户通过标签筛选时调用此方法。
    使用 F 表达式避免竞态条件。
    """
```

**执行流程：**
1. 获取或创建标签统计记录
2. 使用 F 表达式原子性增加 `search_count`
3. 重新计算热度分数
4. 清除对应类型的缓存

##### 3. 更新标签使用统计

```python
@classmethod
def update_tag_usage(cls, tags: List[str], tag_type: str = 'knowledge') -> None:
    """更新标签使用次数
    
    当创建或更新知识库/人设卡时调用此方法。
    """
```

**执行流程：**
1. 遍历标签列表
2. 获取或创建标签统计记录
3. 使用 F 表达式原子性增加 `usage_count`
4. 更新 `last_used_at` 时间戳
5. 重新计算热度分数
6. 清除对应类型的缓存

##### 4. 重建统计数据

```python
@classmethod
def rebuild_statistics(cls) -> Dict[str, int]:
    """重建标签统计数据
    
    扫描所有知识库和人设卡，重新统计标签使用次数。
    用于定时任务或手动触发。
    """
```

**执行流程：**
1. 清空现有统计数据
2. 扫描所有公开的知识库，统计标签使用次数
3. 扫描所有公开的人设卡，统计标签使用次数
4. 批量创建新的统计记录
5. 清除所有类型的缓存

## API 接口

### 获取热门标签

**端点：** `GET /api/content/tags/popular/`

**权限：** 公开访问

**查询参数：**
- `limit` (可选): 返回的标签数量，默认 20，最大 100
- `tag_type` (可选): 标签类型，'knowledge' 或 'persona'，默认 'knowledge'

**响应示例：**

```json
{
    "code": 2000,
    "msg": "获取成功",
    "data": [
        {
            "tag": "阴阳师",
            "usage_count": 5,
            "search_count": 10,
            "hot_score": 7.0
        },
        {
            "tag": "式神录",
            "usage_count": 3,
            "search_count": 5,
            "hot_score": 3.8
        }
    ]
}
```

### 重建标签统计

**端点：** `POST /api/content/tags/rebuild/`

**权限：** 仅管理员

**响应示例：**

```json
{
    "code": 2000,
    "msg": "重建成功",
    "data": {
        "total_tags": 6,
        "knowledge_tags": 3,
        "persona_tags": 3,
        "total_usage": 6
    }
}
```

### 清除标签缓存

**端点：** `POST /api/content/tags/clear_cache/`

**权限：** 仅管理员

**响应示例：**

```json
{
    "code": 2000,
    "msg": "清除成功",
    "data": []
}
```

## 集成说明

### 知识库集成

#### 1. 列表查询时记录搜索统计

在 `KnowledgeBaseViewSet.list()` 方法中：

```python
def list(self, request, *args, **kwargs):
    # 记录标签搜索统计
    tags_param = request.query_params.get('tags', '')
    if tags_param:
        tags = [tag.strip() for tag in tags_param.split(',') if tag.strip()]
        for tag in tags:
            try:
                TagService.increment_search_count(tag, tag_type='knowledge')
            except Exception as e:
                logger.warning(f"记录标签搜索统计失败: {tag}, {e}")
    
    return super().list(request, *args, **kwargs)
```

#### 2. 创建时更新使用统计

在 `KnowledgeBaseViewSet.perform_create()` 方法中：

```python
def perform_create(self, serializer):
    knowledge_base = serializer.save()
    
    # 更新标签使用统计
    if knowledge_base.tags:
        tags = [tag.strip() for tag in knowledge_base.tags.split(',') if tag.strip()]
        try:
            TagService.update_tag_usage(tags, tag_type='knowledge')
        except Exception as e:
            logger.warning(f"更新标签使用统计失败: {e}")
```

#### 3. 更新时更新使用统计

在 `KnowledgeBaseViewSet.perform_update()` 方法中：

```python
def perform_update(self, serializer):
    # 获取旧的标签
    old_tags = set()
    if serializer.instance.tags:
        old_tags = set(tag.strip() for tag in serializer.instance.tags.split(',') if tag.strip())
    
    knowledge_base = serializer.save()
    
    # 获取新的标签
    new_tags = set()
    if knowledge_base.tags:
        new_tags = set(tag.strip() for tag in knowledge_base.tags.split(',') if tag.strip())
    
    # 计算新增的标签
    added_tags = new_tags - old_tags
    if added_tags:
        try:
            TagService.update_tag_usage(list(added_tags), tag_type='knowledge')
        except Exception as e:
            logger.warning(f"更新标签使用统计失败: {e}")
```

### 人设卡集成

人设卡的集成方式与知识库类似，只需将 `tag_type` 参数改为 `'persona'`。

## 性能优化

### 1. 缓存策略

- 热门标签查询结果缓存 1 小时
- 按类型分别缓存，避免缓存失效影响其他类型
- 标签统计更新时自动清除对应类型的缓存

### 2. 数据库优化

- 为常用查询字段创建索引：`tag`, `tag_type`, `hot_score`, `usage_count`
- 使用 F 表达式进行原子性更新，避免竞态条件
- 批量创建统计记录，减少数据库操作次数

### 3. 查询优化

- 默认按热度分数和使用次数排序
- 限制返回结果数量（默认 20，最大 100）
- 使用 `select_related` 和 `prefetch_related` 优化关联查询

## 监控和维护

### 定时任务

建议配置 Celery 定时任务，定期重建标签统计数据：

```python
from celery import shared_task
from mainotebook.content.services.tag_service import TagService

@shared_task
def rebuild_tag_statistics():
    """重建标签统计数据"""
    result = TagService.rebuild_statistics()
    logger.info(f"标签统计重建完成: {result}")
    return result
```

**执行频率建议：** 每天凌晨执行一次

### 缓存监控

监控缓存命中率，确保缓存策略有效：

```python
# 在 get_popular_tags 方法中添加监控

    logger.debug(f"标签缓存命中: {tag_type}")
```

### 数据清理

定期清理长期未使用的标签统计记录：

```python
from django.utils import timezone
from datetime import timedelta

# 删除 6 个月未使用的标签
threshold = timezone.now() - timedelta(days=180)
TagStatistics.objects.filter(
    last_used_at__lt=threshold,
    usage_count=0,
    search_count=0
).delete()
```

### 自定义热度分数算法

如果需要调整热度分数的计算方式，修改 `calculate_hot_score` 方法：

```python
def calculate_hot_score(self) -> float:
    """自定义热度分数计算
    
    可以考虑：
    - 时间衰减：最近使用的标签权重更高
    - 用户反馈：结合用户评分
    - 内容质量：结合内容的审核状态
    """
    # 基础分数
    base_score = self.usage_count * 0.6 + self.search_count * 0.4
    
    # 时间衰减（可选）
    if self.last_used_at:
        days_ago = (timezone.now() - self.last_used_at).days
        decay_factor = 1.0 / (1.0 + days_ago / 30.0)  # 30天衰减
        base_score *= decay_factor
    
    return base_score
```

## 故障排查

### 常见问题

1. **标签不显示**
   - 检查标签统计数据是否存在
   - 检查 `tag_type` 参数是否正确
   - 检查缓存是否正常工作

2. **热度分数不更新**
   - 检查 `update_hot_score()` 是否被调用
   - 检查数据库更新是否成功
   - 清除缓存后重新查询

3. **缓存不生效**
   - 检查 Redis 连接是否正常
   - 检查缓存键是否正确
   - 检查缓存过期时间设置
