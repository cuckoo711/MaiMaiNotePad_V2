# 人设卡上传功能部署指南

## 概述

本文档提供人设卡上传功能的完整部署指南，包括数据库迁移、Redis 配置、Celery 配置等。

## 前置条件

### 系统要求

- Python 3.8+
- PostgreSQL 12+ 或 MySQL 5.7+
- Redis 5.0+
- Celery 5.0+

### 依赖检查

确保以下服务正常运行：

```bash
# 检查 PostgreSQL
psql -U mai_notebook -d mai_notebook -c "SELECT version();"

# 检查 Redis
redis-cli ping

# 检查 Python 环境
conda activate mai_notebook
python --version
```

## 部署步骤

### 1. 数据库迁移

人设卡上传功能需要创建两个新表和一个新字段：

- `mainotebook_content_persona_card_config`：人设卡配置项表
- `mainotebook_content_sensitive_info_confirmation`：敏感信息确认记录表
- `PersonaCard.is_deleted`：软删除标记字段

#### 执行迁移

```bash
# 激活 Conda 环境
conda activate mai_notebook

# 进入项目目录
cd backend

# 检查待执行的迁移
python manage.py showmigrations content

# 执行迁移
python manage.py migrate content

# 验证迁移结果
python manage.py showmigrations content
```

#### 预期输出

```
content
 [X] 0001_initial
 [X] 0002_initial
 [X] 0003_alter_reviewreport_decision
 [X] 0004_aimodel_enable_thinking
 [X] 0005_alter_reviewreport_decision

 [X] 0008_add_tag_type_to_tag_statistics

如果需要回滚迁移：

```

### 2. Redis 配置

人设卡上传功能使用 Redis 存储：

- 上传限额计数（键名：`persona_upload_quota:{user_id}`）
- 验证码重试次数（键名：`captcha_retry:{user_id}`）
- 验证码冷却期（键名：`captcha_cooldown:{user_id}`）

#### 配置检查

确保 `backend/conf/env.py` 中的 Redis 配置正确：

```python
REDIS_DB = 1
CELERY_BROKER_DB = 3
REDIS_PASSWORD = "your_redis_password"
REDIS_HOST = "127.0.0.1"
REDIS_URL = f"redis://:{REDIS_PASSWORD}@{REDIS_HOST}:6379"
```

#### 测试 Redis 连接

```bash
# 使用 Python 测试 Redis 连接
python manage.py shell

>>> from django.core.cache import cache
>>> cache.set('test_key', 'test_value', 60)
>>> cache.get('test_key')
'test_value'
>>> exit()
```

#### Redis 内存配置

建议为人设卡上传功能预留至少 **100MB** Redis 内存：

```bash
# 编辑 Redis 配置文件
sudo nano /etc/redis/redis.conf

# 设置最大内存（示例：2GB）
maxmemory 2gb

# 设置内存淘汰策略
maxmemory-policy allkeys-lru

# 重启 Redis
sudo systemctl restart redis
```

#### 监控 Redis 使用情况

```bash
# 查看 Redis 内存使用
redis-cli info memory

# 查看人设卡上传相关的键
redis-cli --scan --pattern "persona_upload_quota:*"
redis-cli --scan --pattern "captcha_retry:*"
redis-cli --scan --pattern "captcha_cooldown:*"
```

### 3. Celery 配置

人设卡上传功能使用 Celery 执行异步审核任务。

#### 启动 Celery Worker

```bash
# 激活 Conda 环境
conda activate mai_notebook

# 进入项目目录
cd backend

# 启动 Celery Worker（开发环境）
celery -A application worker -l info

# 启动 Celery Worker（生产环境，使用 systemd）
sudo systemctl start celery-worker
```

#### 验证 Celery 状态

```bash
# 检查 Celery Worker 状态
celery -A application inspect active

# 检查 Celery 队列
celery -A application inspect active_queues

# 检查已注册的任务
celery -A application inspect registered
```

#### Celery 配置文件

确保 `backend/application/celery.py` 配置正确：

```python
# Celery 配置
app.conf.update(
    broker_url=REDIS_URL,
    result_backend=REDIS_URL,
    task_serializer='json',
    accept_content=['json'],
    result_serializer='json',
    timezone='UTC',
    enable_utc=True,
)
```

#### 测试异步任务

```bash
# 使用 Python 测试异步任务
python manage.py shell

>>> from mainotebook.content.tasks import auto_review_task
>>> result = auto_review_task.delay('test_id', 'persona')
>>> result.ready()
True
>>> exit()
```

## 生产环境部署

### 1. 使用 Gunicorn 运行应用

```bash
# 安装 Gunicorn
pip install gunicorn

# 启动应用
gunicorn application.wsgi:application \
  --bind 0.0.0.0:8000 \
  --workers 4 \
  --timeout 120 \
  --access-logfile logs/access.log \
  --error-logfile logs/error.log
```

### 2. 使用 Systemd 管理 Celery

创建 `/etc/systemd/system/celery-worker.service`：

```ini
[Unit]
Description=Celery Worker
After=network.target redis.service postgresql.service

[Service]
Type=forking
User=www-data
Group=www-data
WorkingDirectory=/path/to/backend
Environment="PATH=/path/to/conda/envs/mai_notebook/bin"
ExecStart=/path/to/conda/envs/mai_notebook/bin/celery -A application worker -l info --logfile=/path/to/logs/celery/worker.log --pidfile=/var/run/celery/worker.pid
ExecStop=/bin/kill -s TERM $MAINPID
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable celery-worker
sudo systemctl start celery-worker
sudo systemctl status celery-worker
```

### 3. 配置 Nginx

添加文件上传大小限制：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 文件上传大小限制（2MB）
    client_max_body_size 2M;

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        
        # 上传超时设置
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }

    location /media/ {
        alias /path/to/backend/media/;
        expires 30d;
    }
}
```

重启 Nginx：

```bash
sudo nginx -t
sudo systemctl restart nginx
```

## 故障排查

### 问题 1：迁移失败

**症状**：执行 `python manage.py migrate` 时报错

**解决方法**：

```bash
# 检查数据库连接
python manage.py dbshell

# 检查迁移状态
python manage.py showmigrations

# 如果迁移文件损坏，重新生成
python manage.py makemigrations content

# 强制执行迁移
python manage.py migrate content --fake-initial
```

### 问题 2：Redis 连接失败

**症状**：上传限额检查失败，日志显示 Redis 连接错误

**解决方法**：

```bash
# 检查 Redis 服务状态
sudo systemctl status redis

# 检查 Redis 配置
redis-cli ping

# 检查 Redis 密码
redis-cli -a YOUR_PASSWORD ping

# 重启 Redis
sudo systemctl restart redis
```

### 问题 3：Celery 任务不执行

**症状**：人设卡提交审核后，审核状态一直为待审核

**解决方法**：

```bash
# 检查 Celery Worker 状态
celery -A application inspect active

# 检查任务队列
celery -A application inspect active_queues

# 重启 Celery Worker
sudo systemctl restart celery-worker

# 查看 Celery 日志
tail -f backend/logs/celery/worker.log
```

### 问题 4：文件上传失败

**症状**：上传文件时返回 413 错误或文件保存失败

**解决方法**：

```bash
# 检查 Nginx 配置
sudo nginx -t

# 检查文件存储目录权限
ls -la backend/media/persona_cards/

# 修改权限
sudo chown -R www-data:www-data backend/media/persona_cards/
sudo chmod -R 755 backend/media/persona_cards/

# 检查磁盘空间
df -h
```

### 问题 5：验证码验证失败

**症状**：用户输入正确的验证码但仍然提示错误

**解决方法**：

```bash
# 检查验证码配置
python manage.py shell

>>> from captcha.models import CaptchaStore
>>> CaptchaStore.objects.all()
>>> exit()

# 清理过期验证码
python manage.py shell

>>> from captcha.models import CaptchaStore
>>> CaptchaStore.remove_expired()
>>> exit()
```

## 回滚计划

如果部署后发现严重问题，可以按以下步骤回滚：

### 1. 停止服务

```bash
# 停止应用
sudo systemctl stop gunicorn

# 停止 Celery
sudo systemctl stop celery-worker
```

### 2. 回滚数据库

```bash
# 回滚迁移
python manage.py migrate content 0008

# 备份当前数据（可选）
pg_dump -U mai_notebook mai_notebook > backup_before_rollback.sql
```

### 3. 回滚代码

```bash
# 切换到上一个版本
git checkout <previous_commit>

# 重启服务
sudo systemctl start gunicorn
sudo systemctl start celery-worker
```

### 4. 清理 Redis

```bash
# 清理人设卡上传相关的 Redis 键
redis-cli --scan --pattern "persona_upload_quota:*" | xargs redis-cli del
redis-cli --scan --pattern "captcha_retry:*" | xargs redis-cli del
redis-cli --scan --pattern "captcha_cooldown:*" | xargs redis-cli del
```

## 安全建议

1. **文件上传安全**
   - 限制文件大小（1KB - 2MB）
   - 验证文件类型和编码
   - 使用唯一文件名存储

2. **敏感信息保护**
   - 记录所有敏感信息确认操作
   - 保存用户 IP 地址
   - 定期审计确认记录

3. **访问控制**
   - 只有登录用户可以上传
   - 只有上传者可以编辑和删除
   - 只有公开且已审核的人设卡可以下载

4. **限流保护**
   - 每日上传限额（10 次）
   - 验证码重试限制（10 次）
   - 验证码冷却期（1 分钟）

## 性能优化建议

1. **数据库优化**
   - 定期清理已删除的人设卡（软删除）
   - 定期清理过期的验证码记录
   - 监控慢查询并添加索引

2. **Redis 优化**
   - 设置合理的内存限制
   - 使用 LRU 淘汰策略
   - 定期清理过期键

3. **文件存储优化**
   - 定期清理未关联的文件
   - 考虑使用对象存储（如 S3）
   - 启用 CDN 加速下载

4. **Celery 优化**
   - 增加 Worker 数量
   - 使用任务优先级
   - 监控任务执行时间
