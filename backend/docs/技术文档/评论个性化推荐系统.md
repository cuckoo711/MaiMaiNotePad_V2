# 评论个性化推荐系统

## 概述

MaiMaiNotePad 的评论区采用了先进的个性化推荐算法，为每个用户提供定制化的内容排序，同时保证内容多样性和新鲜度。

## 核心特性

### 1. 五层评分机制

#### 1.1 基础热度分数（30%权重）
- **双阶段时间衰减**：
  - 0-24小时：黄金期，不衰减（time_decay = 1.0）
  - 24小时-7天：缓慢衰减，每48小时衰减50%
  - 7天以上：快速衰减，衰减率×0.1
  
- **互动分数**：
  ```python
  engagement_score = 点赞数 × 2.0 + 回复数 × 1.5 - 点踩数 × 0.5
  ```

- **质量加成**：
  ```python
  quality_bonus = min(engagement_score × 0.1, 10)
  ```

- **最终热度**：
  ```python
  base_hot_score = engagement_score × time_decay + quality_bonus + 10
  ```

#### 1.2 用户兴趣权重（70%权重）

**作者偏好**：
- 用户点赞过的评论作者：权重×1.5
- 用户回复过的评论作者：权重×1.5
- 基于历史互动数据自动学习

**内容相似度**（30%关键词 + 70%向量）：
- 关键词匹配：使用 jieba 分词提取关键词，计算 Jaccard 相似度
- 向量相似度：使用 BGE-M3 嵌入模型（1024维），计算余弦相似度
- 综合相似度最多提升权重50%

#### 1.3 多样性因子

- 最近浏览过的作者：权重×0.7
- 避免"信息茧房"，保证内容多样性
- 使用 Redis 缓存浏览历史（24小时过期）

#### 1.4 向量嵌入（BGE-M3）

- **模型**：BAAI/bge-m3（硅基流动免费模型）
- **维度**：1024维
- **上下文**：支持8K token
- **功能**：密集检索、多向量检索、稀疏检索
- **语言**：支持100+种语言，包括中文

#### 1.5 重排序（BGE-Reranker）

- **模型**：BAAI/bge-reranker-v2-m3
- **用途**：对候选集进行精细重排序
- **范围**：仅对前30条候选评论重排序（平衡效果和性能）
- **输出**：直接输出相似度分数

### 2. 技术实现

#### 2.1 架构设计

```
┌─────────────────────────────────────────────────┐
│           CommentService (业务层)                │
│  - get_comments_tree(): 获取评论树（分页）        │
│  - create_comment(): 创建评论                    │
│  - react_comment(): 点赞/点踩                    │
└─────────────────┬───────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────┐
│      RecommendationService (推荐层)              │
│  - get_user_interests(): 获取用户兴趣（缓存）     │
│  - calculate_content_similarity(): 计算相似度     │
│  - rerank_comments(): 重排序                     │
│  - record_user_view(): 记录浏览历史              │
└─────────────────┬───────────────────────────────┘
                  │
        ┌─────────┴─────────┐
        ▼                   ▼
┌──────────────┐    ┌──────────────┐
│  Redis 缓存   │    │  BGE-M3 API  │
│  - 用户兴趣   │    │  - 文本嵌入   │
│  - 浏览历史   │    │  - 重排序     │
└──────────────┘    └──────────────┘
```

#### 2.2 缓存策略

**用户兴趣缓存**：
- 键：`user_interests:{user_id}`
- 过期时间：1小时
- 内容：喜欢的作者、关键词、兴趣向量

**浏览历史缓存**：
- 键：`user_views:{user_id}`
- 过期时间：24小时
- 内容：最近浏览的50个作者ID

**缓存失效**：
- 用户点赞评论时：清除兴趣缓存
- 用户回复评论时：清除兴趣缓存
- 自动过期：1小时后重新计算

#### 2.3 性能优化

1. **数据库优化**：
   - 使用 `select_related` 预加载关联数据
   - 使用 `annotate` 聚合回复数
   - 限制查询数量（点赞100条，回复50条）

2. **缓存优化**：
   - Redis 缓存用户兴趣（1小时）
   - Redis 缓存浏览历史（24小时）
   - 避免重复计算

3. **API 优化**：
   - 批量调用嵌入API
   - 仅对前30条候选重排序
   - 异步记录浏览历史

4. **分词优化**：
   - 使用 jieba 分词（快速）
   - TF-IDF 提取关键词（topK=10）
   - 缓存分词结果

### 3. 用户体验

#### 3.1 已登录用户

**个性化排序**：
- 看到更多感兴趣的内容
- 喜欢的作者优先展示
- 相似主题的评论聚合

**多样性保证**：
- 避免重复看到相同作者
- 新评论有曝光机会
- 热门评论不会被埋没

#### 3.2 未登录用户

**纯热度排序**：
- 看到最热门的讨论
- 不会因为没有数据而体验下降
- 快速了解社区热点

### 4. 数据积累

系统会自动积累用户行为数据：

| 行为类型 | 权重 | 用途 |
|---------|------|------|
| 点赞评论 | 2 | 作者偏好、内容兴趣 |
| 回复评论 | 3 | 强兴趣信号 |
| 浏览评论 | 1 | 多样性控制 |

随着数据积累，推荐效果会越来越好！

### 5. API 接口

#### 5.1 获取评论列表（分页）

```http
GET /api/content/comments/?target_id={id}&target_type={type}&page={page}&page_size={size}
```

**参数**：
- `target_id`：目标ID（必需）
- `target_type`：目标类型（knowledge/persona，必需）
- `page`：页码（默认1）
- `page_size`：每页数量（默认10）

**响应**：
```json
{
  "code": 2000,
  "msg": "获取成功",
  "data": [...],
  "total": 100,
  "page": 1,
  "page_size": 10,
  "has_more": true
}
```

#### 5.2 加载更多回复

```http
GET /api/content/comments/{comment_id}/replies/?page={page}&page_size={size}
```

**参数**：
- `comment_id`：父评论ID（必需）
- `page`：页码（默认1）
- `page_size`：每页数量（默认10）

### 6. 配置参数

可在 `recommendation_service.py` 中调整的参数：

```python
# 缓存过期时间
CACHE_TTL_USER_INTERESTS = 3600  # 用户兴趣：1小时
CACHE_TTL_USER_VIEWS = 86400     # 浏览历史：24小时

# 数据量限制
liked_comments_limit = 100       # 点赞历史
replied_comments_limit = 50      # 回复历史
recent_views_limit = 50          # 浏览历史

# 权重参数
author_weight = 1.5              # 喜欢的作者权重
content_similarity_weight = 0.5  # 内容相似度权重
diversity_factor = 0.7           # 多样性因子

# 排序权重
interest_weight = 0.7            # 兴趣权重70%
hot_weight = 0.3                 # 热度权重30%
```
