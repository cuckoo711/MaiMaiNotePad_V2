# 首页重构方案

## 一、重构目标

将管理后台首页从"管理向"改为"用户向",展示对普通用户有价值的统计信息和内容推荐。

## 二、页面布局

### 2.1 顶部统计卡片(4个)

1. **知识库总数** - 平台公开的知识库数量
2. **人设卡总数** - 平台公开的人设卡数量  
3. **我的上传** - 当前用户上传的内容数量
4. **我的下载** - 当前用户下载的内容数量

### 2.2 左侧区域

#### 2.2.1 月度趋势图(折线图)
- 标题: "内容上传趋势"
- X轴: 最近12个月
- Y轴: 上传数量
- 两条线:
  - 知识库上传数量(橙色)
  - 人设卡上传数量(紫色)

#### 2.2.2 热门内容排行(柱状图)
- 标题: "热门内容下载排行"
- 展示下载量前10的内容
- 区分知识库和人设卡

### 2.3 右侧区域

#### 2.3.1 欢迎横幅
- 显示用户头像和欢迎信息
- 保持原有设计

#### 2.3.2 系统通知
- 显示最新的系统消息
- 保持原有功能

#### 2.3.3 快捷导航
- 移除管理功能(角色、部门、配置等)
- 改为用户功能:
  - 浏览知识库
  - 浏览人设卡
  - 我的上传
  - 我的下载
  - 消息中心
  - 个人设置

## 三、后端API设计

### 3.1 API端点

基础路径: `/api/system/home_stats/`

#### 3.1.1 平台统计
```
GET /api/system/home_stats/platform_stats/
```

响应:
```json
{
  "code": 2000,
  "data": {
    "total_knowledge": 1234,
    "total_persona": 567,
    "total_users": 890,
    "total_downloads": 5678
  }
}
```

#### 3.1.2 个人统计
```
GET /api/system/home_stats/my_stats/
```

响应:
```json
{
  "code": 2000,
  "data": {
    "my_uploads": 10,
    "my_downloads": 25,
    "my_comments": 15,
    "my_pending": 2
  }
}
```

#### 3.1.3 热门内容
```
GET /api/system/home_stats/hot_content/
```

响应:
```json
{
  "code": 2000,
  "data": {
    "hot_knowledge": [
      {
        "id": "uuid",
        "name": "知识库名称",
        "description": "描述",
        "download_count": 100
      }
    ],
    "hot_persona": [...]
  }
}
```

#### 3.1.4 最新内容
```
GET /api/system/home_stats/recent_content/
```

响应:
```json
{
  "code": 2000,
  "data": {
    "recent_knowledge": [...],
    "recent_persona": [...]
  }
}
```

#### 3.1.5 月度趋势
```
GET /api/system/home_stats/monthly_trend/
```

响应:
```json
{
  "code": 2000,
  "data": {
    "months": ["1月", "2月", ..., "12月"],
    "knowledge_counts": [10, 15, 20, ...],
    "persona_counts": [5, 8, 12, ...]
  }
}
```

## 六、注意事项

1. **权限控制**: 平台统计、热门内容、最新内容允许所有人访问;个人统计需要登录
2. **性能优化**: 考虑添加缓存,避免频繁查询数据库
3. **数据安全**: 不暴露敏感的管理数据
4. **响应式设计**: 确保在不同屏幕尺寸下正常显示
5. **国际化**: 所有文本支持多语言

## 七、后续优化

2. 添加数据刷新按钮
3. 支持自定义时间范围
4. 添加更多维度的统计(评论数、点赞数等)
5. 个性化推荐内容
