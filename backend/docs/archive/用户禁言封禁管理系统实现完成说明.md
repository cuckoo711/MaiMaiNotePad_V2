**实现内容**:

## 技术实现

### 架构设计

采用经典的分层架构：

```
API层 (ViewSet)
    ↓
服务层 (Services)
    ↓
数据层 (Models)
    ↓
工具层 (Utils)
```

### 关键技术

1. **数据库事务**: 所有涉及多表操作的业务逻辑都使用数据库事务，确保数据一致性

2. **Celery定时任务**: AI自动禁言使用Celery创建一次性定时任务，实现自动解封

3. **权限验证**: 实现了严格的权限分级验证机制

4. **通知降级**: 通知发送失败不影响主流程，采用降级策略

5. **幂等性**: 解除操作支持幂等，重复操作不会报错

6. **索引优化**: 在关键查询字段上创建索引，提升查询性能

### 代码规范

- ✅ 所有函数包含类型提示

- ✅ 代码标识符使用英文（snake_case/PascalCase）
- ✅ 注释和文档使用简体中文
- ✅ 错误消息使用简体中文
- ✅ 遵循PEP 8标准

## 配置说明

### 环境变量配置

在 `.env` 文件中添加或修改以下配置：

```bash
# AI自动禁言配置
AUTO_MUTE_WINDOW_HOURS=24      # 滚动时间窗口（小时），默认24
AUTO_MUTE_THRESHOLD=6          # 触发阈值（次数），默认6
AUTO_MUTE_DURATION_HOURS=24    # 自动禁言时长（小时），默认24
```

### 初始化数据

执行以下命令初始化菜单、角色和权限：

```bash
python manage.py init_menu
python manage.py init_role
python manage.py init_permissions
```

### Celery服务

确保Celery服务正常运行：

```bash
# 启动Worker
celery -A backend worker -l info

# 启动Beat（定时任务调度器）
celery -A backend beat -l info
```

## 使用说明

### 管理员操作

1. **分配权限**: 为管理员分配 `moderation_admin` 角色
2. **访问菜单**: 登录后台，进入"用户管理"菜单
3. **执行操作**: 根据需要执行禁言、封禁或批量操作
4. **查看日志**: 通过操作日志追溯所有操作记录

### 用户体验

1. **被禁言**: 无法发表评论，显示禁言提示
2. **被封禁**: 无法登录系统，显示封禁提示
3. **收到通知**: 通过消息中心和邮件接收通知
4. **自动解除**: 到期后自动解除禁言/封禁

---

## 已知限制

1. **批量操作限制**: 单次批量操作最多支持20个用户
2. **时长格式**: 仅支持预定义的时长格式（h/d/w/m/permanent）
3. **通知降级**: 邮件发送失败时仅记录日志，不重试
4. **测试覆盖**: 当前缺少自动化测试
