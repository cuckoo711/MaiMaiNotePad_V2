# 系统账号前端显示优化说明

## 概述

本文档记录了系统账号在前端页面的显示优化，解决了 ai_reviewer 系统账号被错误显示为"已封禁"状态的问题。

## 优化时间

2026-03-07

## 问题描述

在处罚操作页面中，ai_reviewer 系统账号的"封禁状态"列显示为"已封禁"，这是因为：

1. 系统账号的 `is_active=False` 是为了禁止登录
2. 前端将 `is_active=False` 统一解释为"已封禁"
3. 没有区分系统账号和普通用户的不同语义

这会给管理员造成困惑，误以为系统账号被封禁了。

## 解决方案

### 核心思路

通过 `user_type` 字段区分系统账号和普通用户，在前端显示时采用不同的逻辑：

- **系统账号** (user_type=2): 显示"系统账号"标签
- **普通用户** (user_type=0/1): 根据 `is_active` 显示"已封禁"或"正常"

## 实现内容

### 1. 处罚操作页面

**文件**: `admin_web/src/views/content/moderation/punish/index.vue`

**修改内容**: 在"封禁状态"列的自定义模板中添加系统账号判断

```vue
<template #cell_is_active="scope">
    <!-- 系统账号显示特殊状态 -->
    <el-tag v-if="scope.row.user_type === 2" type="info" size="small">
        系统账号
    </el-tag>
    <!-- 普通用户显示封禁状态 -->
    <el-tag v-else-if="!scope.row.is_active" type="danger" size="small">
        已封禁
    </el-tag>
    <el-tag v-else type="success" size="small">
        正常
    </el-tag>
</template>
```

**效果**:
- 系统账号: 显示灰色"系统账号"标签
- 已封禁用户: 显示红色"已封禁"标签
- 正常用户: 显示绿色"正常"标签

### 2. 用户管理页面

**文件**: `admin_web/src/views/system/user/crud.tsx`

#### 2.1 隐藏删除按钮

**修改内容**: 在删除按钮的 `show` 配置中添加系统账号判断

```typescript
remove: {
    iconRight: 'Delete',
    type: 'text',
    show: compute(({ row }) => {
        // 系统账号不显示删除按钮
        if (row.user_type === 2) {
            return false;
        }
        return auth('user:Delete');
    }),
},
```

**效果**: 系统账号不显示删除按钮

#### 2.2 禁用状态开关

**修改内容**: 在状态列的开关组件中添加禁用逻辑

```typescript
is_active: {
    title: '状态',
    column: {
        component: {
            name: 'fs-dict-switch',
            disabled: compute((context) => {
                // 系统账号禁用开关
                return context.row.user_type === 2;
            }),
            onChange: compute((context) => {
                return () => {
                    // 系统账号不允许修改状态
                    if (context.row.user_type === 2) {
                        return;
                    }
                    api.UpdateObj(context.row).then((res: APIResponseData) => {
                        successMessage(res.msg as string);
                    });
                };
            }),
        },
    },
}
```

**效果**: 
- 系统账号的状态开关被禁用（灰色），无法点击
- 普通用户的状态开关正常工作

## 技术细节

### 判断条件

使用 `user_type === 2` 来判断是否为系统账号：

```typescript
// Vue 模板中
v-if="scope.row.user_type === 2"

// TypeScript 中
if (row.user_type === 2) {
    return false;
}
```

### 标签样式

使用 Element Plus 的 Tag 组件，不同状态使用不同的 type：

- `type="info"`: 灰色，用于系统账号
- `type="danger"`: 红色，用于已封禁
- `type="success"`: 绿色，用于正常状态

### 开关禁用

使用 fast-crud 的 `compute` 函数动态计算 `disabled` 属性：

```typescript
disabled: compute((context) => {
    return context.row.user_type === 2;
})
```

3. 普通用户根据实际状态显示
