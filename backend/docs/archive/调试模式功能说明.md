# 调试模式功能说明

## 完成时间

2026-03-03

## 功能概述

为 `start_backend.sh` 启动脚本添加了 `--debug` 参数，启用后端服务的自动重载功能，方便开发调试。

## 使用方法

### 启动调试模式

```bash
bash start_backend.sh --debug
```

### 正常启动（生产模式）

```bash
bash start_backend.sh
```

## 功能特性

### 调试模式（--debug）

- ✅ 自动监控代码文件变化
- ✅ 检测到修改后自动重启服务
- ✅ 使用单进程模式（workers=1）
- ✅ 适合开发调试使用

### 正常模式（默认）

- ✅ 多进程模式（workers=4）
- ✅ 更高的性能和并发能力
- ✅ 适合生产环境使用

## 实现细节

### 1. 启动脚本修改

在 `start_backend.sh` 中添加了 `--debug` 参数支持：

```bash
# 解析参数
FLAG_DEBUG=false

parse_args() {
    while [[ $# -gt 0 ]]; do
        case "$1" in
            --debug)  FLAG_DEBUG=true ;;
            # ... 其他参数
        esac
        shift
    done
}

start_backend() {
    if [[ "$FLAG_DEBUG" == true ]]; then
        # 调试模式：启用自动重载
        "$PYTHON_CMD" main.py --reload
    else
        # 正常模式
        "$PYTHON_CMD" main.py
    fi
}
```

### 2. main.py 修改

添加了命令行参数解析，支持 `--reload` 参数：

```python
import argparse

parser = argparse.ArgumentParser(description='麦麦笔记本后端服务')
parser.add_argument('--reload', action='store_true', help='启用自动重载（调试模式）')
parser.add_argument('--host', default='0.0.0.0', help='监听地址（默认: 0.0.0.0）')
parser.add_argument('--port', type=int, default=8000, help='监听端口（默认: 8000）')
args = parser.parse_args()

# 自动重载模式下不使用多进程
workers = 1 if args.reload else 4

uvicorn.run(
    "application.asgi:application",
    reload=args.reload,
    host=args.host,
    port=args.port,
    workers=workers,
    log_config=LOGGING,
    timeout_graceful_shutdown=5
)
```

## 监控范围

调试模式会监控整个 `backend/` 目录下的文件变化，包括：

- Python 源代码文件（.py）
- 配置文件
- 模板文件
- 其他项目文件

## 注意事项

1. **性能影响**：调试模式使用单进程，性能低于正常模式，仅用于开发环境
2. **自动重启**：文件修改后服务会自动重启，可能导致正在处理的请求中断
3. **数据库连接**：重启时会重新建立数据库连接
4. **SSE 连接**：重启会断开所有 SSE 长连接，前端需要重新连接

## 其他启动参数

### 完整参数列表

```bash
bash start_backend.sh [选项]

选项:
  --init      首次启动：生成配置 + 数据库迁移 + 初始化数据
  --reset     全量重置：重置数据库/缓存 + 重新配置 + 重新初始化
  --debug     调试模式：启用自动重载（代码修改后自动重启）
  --help      显示帮助信息
```

### 参数组合

可以组合使用多个参数：

```bash
# 首次启动 + 调试模式
bash start_backend.sh --init --debug

# 全量重置 + 调试模式
bash start_backend.sh --reset --debug
```
