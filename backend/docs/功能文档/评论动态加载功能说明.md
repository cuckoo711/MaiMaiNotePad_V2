# 评论动态加载功能说明

## 概述

评论区现已支持动态分页加载，提升大量评论场景下的性能和用户体验。

## 功能特性

### 1. 根评论分页加载

- 每次加载 10 条根评论
- 滚动到底部显示"加载更多评论"按钮
- 支持个性化推荐排序（已登录用户）
- 支持热度排序（未登录用户）

### 2. 二级回复分页加载

- 每条根评论初始显示前 10 条回复
- 当回复数超过 10 条时，显示"展开更多回复"按钮
- 点击按钮加载下一页回复（每次 10 条）
- 显示剩余回复数量提示

### 3. 性能优化

- 按需加载，减少初始加载时间
- 使用 Redis 缓存用户兴趣和浏览历史
- 批量调用嵌入 API，减少网络请求
- 仅对前 30 条候选评论进行重排序

## 前端实现

### API 接口

#### 获取评论列表（分页）

```typescript
import { getComments } from '/@/api/comment';

const response = await getComments(
  targetId,      // 目标 ID
  targetType,    // 目标类型（knowledge/persona）
  page,          // 页码（默认 1）
  pageSize       // 每页数量（默认 10）
);

// 响应格式
{
  code: 2000,
  msg: "获取成功",
  data: [...],           // 评论列表
  total: 100,            // 总评论数
  page: 1,               // 当前页码
  page_size: 10,         // 每页数量
  has_more: true         // 是否还有更多
}
```

#### 加载更多回复

```typescript
import { getReplies } from '/@/api/comment';

const response = await getReplies(
  commentId,     // 父评论 ID
  page,          // 页码（默认 1）
  pageSize       // 每页数量（默认 10）
);

// 响应格式
{
  code: 2000,
  msg: "获取成功",
  data: [...],           // 回复列表
  total: 50,             // 总回复数
  page: 1,               // 当前页码
  page_size: 10,         // 每页数量
  has_more: true         // 是否还有更多
}
```

### 组件使用

#### CommentSection 组件

```vue
<CommentSection
  :target-id="contentId"
  :target-type="'knowledge'"
  :show-header="true"
/>
```

**Props**:
- `target-id`: 目标内容 ID（必需）
- `target-type`: 目标类型，'knowledge' 或 'persona'（必需）
- `show-header`: 是否显示评论区标题（默认 true）

**功能**:
- 自动加载第一页评论
- 显示"加载更多评论"按钮（当有更多评论时）
- 显示评论总数
- 支持发表新评论

#### CommentItem 组件

```vue
<CommentItem
  :comment="comment"
  :target-id="targetId"
  :target-type="targetType"
  :is-reply="false"
  @reply="handleReply"
  @delete="handleDelete"
  @react="handleReact"
  @refresh="loadComments"
/>
```

**Props**:
- `comment`: 评论对象（必需）
- `target-id`: 目标内容 ID（必需）
- `target-type`: 目标类型（必需）
- `is-reply`: 是否为回复（默认 false）

**功能**:
- 显示评论内容和用户信息
- 支持点赞/点踩
- 支持回复
- 支持删除（仅作者）
- 自动加载更多回复（当回复数超过 10 条时）

comments = Comment.objects.filter(
    target_id=target_id,
    target_type=target_type,
    parent__isnull=True,
    is_deleted=False
).select_related('user').exclude(
    moderation_status='rejected'
)

# 使用 annotate 聚合回复数

comments = comments.annotate(
    reply_count=Count('replies', filter=Q(replies__is_deleted=False))

## 用户体验

### 已登录用户

1. **个性化排序**：
   - 优先展示喜欢的作者的评论
   - 推荐相似主题的评论

2. **动态加载**：

3. **智能推荐**：

## 故障排查

### 问题：评论加载缓慢

**可能原因**：
1. 数据库查询未优化
2. 缓存未命中
3. API 调用超时

**解决方案**：
1. 检查数据库索引
2. 检查 Redis 连接
3. 检查嵌入 API 可用性

### 问题：推荐效果不佳

**可能原因**：
1. 用户数据不足
2. 权重参数不合理
3. 缓存数据过期

**解决方案**：
1. 积累更多用户行为数据
2. 调整权重参数（A/B 测试）
3. 清除缓存重新计算

### 问题：展开更多回复不显示

**可能原因**：
1. `reply_total` 字段未返回
2. 前端状态未更新
3. 回复数量计算错误

**解决方案**：
1. 检查序列化器是否包含 `reply_total`
2. 检查 `watch` 是否正确触发
3. 检查数据库查询条件
