# 用户禁言封禁管理使用指南

## 概述

用户禁言封禁管理系统是MaiMaiNotePad平台的核心内容管理功能，为平台管理员提供完整的用户行为管控能力。通过本系统，管理员可以对违规用户进行禁言或封禁处理，维护社区秩序。

## 功能特性

### 核心功能

1. **手动禁言管理** - 管理员可以手动禁言违规用户，禁止其发表评论
2. **手动封禁管理** - 管理员可以封禁严重违规用户，阻止其登录系统
3. **AI自动禁言** - 系统自动识别频繁发布违规评论的用户并自动禁言
4. **批量操作** - 支持批量禁言、封禁和解除操作，提高管理效率
5. **操作日志** - 完整记录所有管理操作，支持审计追溯
6. **通知机制** - 自动向用户发送禁言/封禁通知
7. **数据导出** - 支持导出禁言/封禁记录为CSV或Excel文件

### 权限分级

系统实现了严格的权限分级管理：

- **超级管理员**: 可以操作所有用户（包括普通管理员和普通用户）
- **普通管理员**: 只能操作普通用户，不能操作其他管理员
- **限制规则**: 任何管理员都不能操作自己

## 禁言管理

### 功能说明

禁言是指限制用户发表评论的权限，被禁言的用户无法发表新评论，但可以正常登录和浏览内容。

### 添加禁言

1. 进入"禁言管理"页面
2. 点击"添加禁言"按钮
3. 填写禁言信息：
   - **用户**: 选择或输入要禁言的用户（支持搜索）
   - **时长**: 选择禁言时长
     - 快捷选项: 1小时、3小时、1天、3天、7天、1周、1个月、永久
     - 自定义: 输入数字 + 选择单位（小时/天/周/月）
   - **原因**: 输入禁言原因（必填）
4. 点击"确定"提交

**示例**:
- 禁言3天: 选择"3天"或输入"3"并选择"天"
- 永久禁言: 选择"永久"

### 解除禁言

1. 在禁言列表中找到要解除的用户
2. 点击"解除禁言"按钮
3. 填写解除原因（可选）
4. 点击"确定"

### 修改禁言时长

1. 在禁言列表中找到要修改的用户
2. 点击"修改时长"按钮
3. 选择新的时长
4. 填写修改原因（必填）
5. 点击"确定"

**注意**: 修改时长后，如果是AI自动禁言，将不会自动解封。

### 批量禁言

1. 在禁言列表中勾选要禁言的用户（最多20个）
2. 点击"批量禁言"按钮
3. 填写禁言信息（时长和原因）
4. 系统会显示确认对话框，确认后执行
5. 查看批量操作结果

### 查询和筛选

禁言列表支持多种筛选条件：

- **用户名**: 模糊搜索用户名
- **用户ID**: 精确搜索用户ID
- **状态**: 
  - 禁言中: 当前正在禁言的用户
  - 已解除: 已经解除禁言的用户
- **禁言类型**:
  - 手动禁言: 管理员手动添加的禁言
  - AI自动禁言: 系统自动触发的禁言
- **操作人**: 按执行操作的管理员筛选
- **时间范围**: 按操作时间筛选
- **永久状态**: 筛选永久禁言或定时禁言

### 查看详情

点击"查看详情"按钮可以查看：
- 用户的完整禁言历史
- 所有相关的操作日志
- 操作人信息
- 时长变更记录

### 导出数据

1. 设置筛选条件（可选）
2. 点击"导出"按钮
3. 选择导出格式（CSV或Excel）
4. 系统会下载包含当前筛选结果的文件

## 封禁管理

### 功能说明

封禁是指禁止用户登录系统，被封禁的用户无法登录，也无法进行任何操作。封禁比禁言更严格，适用于严重违规的用户。

### 添加封禁

1. 进入"封禁管理"页面
2. 点击"添加封禁"按钮
3. 填写封禁信息：
   - **用户**: 选择要封禁的用户
   - **时长**: 选择封禁时长（同禁言）
   - **原因**: 输入封禁原因（必填）
4. 点击"确定"提交

### 解除封禁

1. 在封禁列表中找到要解除的用户
2. 点击"解除封禁"按钮
3. 填写解除原因（可选）
4. 点击"确定"

### 修改封禁时长

操作方式同禁言管理。

### 批量封禁

操作方式同批量禁言，最多支持20个用户。

### 查询和筛选

封禁列表支持的筛选条件：
- 用户名
- 用户ID
- 状态（封禁中/已解除）
- 操作人
- 时间范围
- 永久状态

### 导出数据

操作方式同禁言管理。

## AI自动禁言

### 功能说明

AI自动禁言是系统根据用户的违规行为自动触发的禁言机制。当用户在一定时间内频繁发布违规评论时，系统会自动禁言该用户。

### 触发机制

**默认规则**:
- 时间窗口: 24小时
- 触发阈值: 评论被AI拒绝6次
- 自动禁言时长: 24小时

**工作流程**:
1. 用户发表评论
2. AI审核系统检测评论内容
3. 如果评论违规，拒绝发布并记录
4. 系统统计用户在24小时内的拒绝次数
5. 当拒绝次数达到6次时，自动触发禁言
6. 禁言24小时后自动解除

### 查看AI自动禁言列表

1. 进入"AI自动禁言"页面
2. 查看所有AI自动触发的禁言记录

列表显示信息：
- 用户信息
- 禁言原因（通常是"24小时内评论被拒绝X次"）
- 禁言时间
- 剩余时长
- 自动解封状态：
  - **等待中**: 将在到期时自动解封
  - **已解封**: 已经自动解封
  - **已被人工修改**: 管理员修改过，不会自动解封

### 人工干预

管理员可以对AI自动禁言进行干预：

1. **解除禁言**: 提前解除AI自动禁言
2. **修改时长**: 延长或缩短禁言时长

**注意**: 任何人工修改都会取消自动解封机制。

### 评论拒绝警告

当用户评论被拒绝但未达到自动禁言阈值时，系统会发送警告通知：

**通知内容**:
- 评论被拒绝的原因
- 当前已被拒绝的次数
- 距离自动禁言还剩多少次机会
- 自动禁言的后果说明

**示例**:
```
您的评论因"包含不当内容"被拒绝。
这是您在24小时内第3次评论被拒绝。
再被拒绝3次将被自动禁言24小时。
请注意您的言论，遵守社区规范。
```

## 通知机制

### 用户通知

系统会在以下情况自动向用户发送通知：

#### 1. 禁言通知

**发送时机**: 用户被禁言时

**通知方式**: 消息中心 + 邮件

**通知内容**:
```
禁言通知

您的账号已被禁言。

禁言原因: 发布违规内容
禁言类型: 手动禁言
禁言时长: 3天
截止时间: 2026-03-10 15:30:00

在禁言期间，您将无法发表评论。
如有疑问，请联系管理员。
```

#### 2. 解除禁言通知

**发送时机**: 禁言被解除时

**通知方式**: 消息中心 + 邮件

**通知内容**:
```
解除禁言通知

您的禁言状态已被解除。

解除原因: 申诉成功

您现在可以正常发表评论了。
```

#### 3. 封禁通知

**发送时机**: 用户被封禁时

**通知方式**: 消息中心 + 邮件

**通知内容**:
```
封禁通知

您的账号已被封禁。

封禁原因: 严重违规行为
封禁时长: 7天
截止时间: 2026-03-14 15:30:00

在封禁期间，您将无法登录系统。
如有疑问，请联系管理员。
```

#### 4. 解除封禁通知

**发送时机**: 封禁被解除时

**通知方式**: 消息中心 + 邮件

**通知内容**:
```
解除封禁通知

您的封禁状态已被解除。

解除原因: 申诉成功

您现在可以正常登录系统了。
```

#### 5. 评论拒绝警告

**发送时机**: 评论被AI拒绝但未达到自动禁言阈值时

**通知方式**: 仅消息中心

**通知内容**:
```
评论审核未通过提醒

您的评论因"包含不当内容"被拒绝。

当前统计: 24小时内已被拒绝3次
剩余机会: 还有3次机会
警告: 再被拒绝3次将被自动禁言24小时

请注意您的言论，遵守社区规范。
```

## 用户体验

### 被禁言用户的体验

1. **尝试发表评论时**:
   - 系统会阻止评论发布
   - 显示错误提示：
     ```
     您已被禁言至2026-03-10 15:30:00，无法发表评论。
     禁言原因：发布违规内容
     ```

2. **永久禁言**:
   - 显示错误提示：
     ```
     您已被永久禁言，无法发表评论。
     禁言原因：严重违规
     ```

3. **禁言已过期**:
   - 系统自动解除禁言状态
   - 用户可以正常发表评论

### 被封禁用户的体验

1. **尝试登录时**:
   - 系统会阻止登录
   - 显示错误提示：
     ```
     您的账号已被封禁至2026-03-14 15:30:00，无法登录。
     封禁原因：严重违规行为
     ```

2. **永久封禁**:
   - 显示错误提示：
     ```
     您的账号已被永久封禁，无法登录。
     封禁原因：严重违规
     ```

3. **封禁已过期**:
   - 系统自动解除封禁状态
   - 用户可以正常登录

## 最佳实践

### 禁言管理建议

1. **明确原因**: 禁言时务必填写清晰的原因，便于用户理解和申诉
2. **合理时长**: 根据违规程度选择合适的时长
   - 轻微违规: 1-3小时
   - 一般违规: 1-3天
   - 严重违规: 7天-1个月
   - 极其严重: 永久禁言
3. **及时处理**: 发现违规行为后及时处理，维护社区秩序
4. **记录详细**: 在操作日志中记录详细信息，便于后续追溯

### 封禁管理建议

1. **谨慎使用**: 封禁是最严厉的处罚，应谨慎使用
2. **适用场景**:
   - 恶意注册大量账号
   - 严重违反社区规范
   - 多次违规且屡教不改
   - 涉及违法违规内容
3. **提供申诉渠道**: 确保用户有申诉的途径

### 批量操作建议

1. **二次确认**: 批量操作前务必仔细核对用户列表
2. **数量限制**: 单次操作不超过20个用户
3. **统一原因**: 批量操作时使用统一的原因描述
4. **分批处理**: 如需操作大量用户，建议分批进行

### AI自动禁言配置建议

1. **合理阈值**: 根据平台实际情况调整触发阈值
   - 严格管理: 阈值设为3-5次
   - 一般管理: 阈值设为6-8次
   - 宽松管理: 阈值设为10次以上

2. **时间窗口**: 建议设置为24小时，既能有效识别违规行为，又不会误伤正常用户

3. **禁言时长**: 建议设置为24-48小时，给用户改正的机会

4. **定期审查**: 定期查看AI自动禁言列表，评估AI审核效果

## 常见问题

### Q1: 如何修改AI自动禁言的配置？

**A**: 需要修改环境变量：

```bash
# 在 .env 文件中添加或修改
AUTO_MUTE_WINDOW_HOURS=24      # 滚动时间窗口（小时）
AUTO_MUTE_THRESHOLD=6          # 触发阈值
AUTO_MUTE_DURATION_HOURS=24    # 自动禁言时长（小时）
```

修改后需要重启服务。

### Q2: 为什么我不能禁言某个用户？

**A**: 可能的原因：
1. 您没有 `moderation_admin` 权限
2. 您是普通管理员，尝试禁言其他管理员
3. 您尝试禁言自己

### Q3: AI自动禁言会自动解除吗？

**A**: 是的，AI自动禁言会在到期时自动解除，除非：
1. 管理员手动解除了禁言
2. 管理员修改了禁言时长

任何人工修改都会取消自动解封机制。

### Q4: 批量操作失败了怎么办？

**A**: 批量操作会返回详细的结果，包括：
- 成功的用户列表
- 失败的用户列表及失败原因

您可以根据失败原因，对失败的用户单独处理。

### Q5: 如何查看某个用户的完整操作历史？

**A**: 
1. 在禁言或封禁列表中找到该用户
2. 点击"查看详情"按钮
3. 系统会显示该用户的所有相关操作记录

### Q6: 用户申诉后如何处理？

**A**:
1. 审查用户的申诉理由
2. 查看用户的操作历史和违规记录
3. 如果申诉合理，使用"解除禁言"或"解除封禁"功能
4. 填写解除原因，如"申诉成功"

### Q7: 如何导出所有永久禁言的用户？

**A**:
1. 进入禁言管理页面
2. 在筛选条件中选择"永久状态"为"是"
3. 点击"导出"按钮
4. 选择导出格式（CSV或Excel）

### Q8: 禁言和封禁可以同时生效吗？

**A**: 可以。禁言和封禁是独立的两个状态：
- 禁言：限制发表评论
- 封禁：限制登录系统

如果用户同时被禁言和封禁，封禁的限制更严格（无法登录）。
