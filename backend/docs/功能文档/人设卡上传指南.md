# 人设卡上传指南

## 概述

人设卡上传功能允许注册用户上传和管理 bot_config.toml 配置文件。用户可以填写基本信息、上传配置文件、在线编辑配置、管理敏感信息，并选择公开或私有。公开的人设卡需要经过 AI 自动审核和可能的人工审核流程，审核通过后其他用户可以下载使用。

## 功能特性

### 核心功能

1. **配置文件上传与解析**：支持 bot_config.toml 文件的上传、验证和结构化解析
2. **在线配置编辑**：提供可视化的配置编辑界面，支持多种数据类型的编辑
3. **敏感信息检测与确认**：自动检测配置中的敏感信息（5-11 位数字），要求用户确认
4. **审核流程**：公开人设卡需经过 AI 自动审核和可能的人工审核
5. **版本管理**：支持配置项的修改历史追踪
6. **下载管理**：支持已审核人设卡的导出和下载

### 上传限额

- 每个用户每天最多上传 **10 个**人设卡
- 限额在 **UTC 零点**自动重置
- 超过限额后需等待次日才能继续上传

## API 端点

### 1. 解析 TOML 文件

**端点**：`POST /api/content/persona/parse-toml/`

**说明**：解析上传的 bot_config.toml 文件，返回结构化数据和敏感信息检测结果。此端点不创建数据库记录，仅用于前端预览。

**请求参数**：
- `file`（File，必填）：bot_config.toml 文件

**文件要求**：
- 文件名必须为 `bot_config.toml`
- 文件大小：1KB - 2MB
- MIME 类型：`text/plain` 或 `application/toml`
- 编码：UTF-8
- 必须包含 `version` 字段（在顶层或 `inner.meta.card` 中）

**响应示例**：
```json
{
  "code": 2000,
  "msg": "解析成功",
  "data": {
    "sections": [
      {
        "name": "inner.meta.card",
        "comment": "人设卡元信息",
        "items": [
          {
            "key": "version",
            "value": "1.0",
            "type": "string",
            "comment": "版本号"
          }
        ]
      }
    ],
    "version": "1.0",
    "sensitive_info": [
      {
        "section": "contact",
        "key": "qq",
        "value": "12345678",
        "matches": ["12345678"],
        "path": "contact.qq"
      }
    ]
  }
}
```

**错误响应**：
- `400`：文件验证失败（文件名错误、大小超限、格式错误等）
- `400`：TOML 解析失败（语法错误、缺少 version 字段等）

### 2. 创建人设卡及配置

**端点**：`POST /api/content/persona/create-with-config/`

**说明**：创建人设卡并保存配置项。支持上传文件、配置敏感信息确认。

**权限**：需要登录，未被封禁

**请求参数**：
```json
{
  "name": "我的人设卡",
  "description": "这是一个测试人设卡，包含完整的配置信息",
  "copyright_owner": "张三",
  "tags": "测试,示例",
  "is_public": false,
  "configs": [
    {
      "section_name": "inner.meta.card",
      "key_name": "version",
      "value": "1.0",
      "data_type": "string",
      "description": "版本号"
    }
  ],
  "sensitive_confirmation": {
    "text": "我已确认该文件在第 10 行的内容不涉及个人隐私信息",
    "locations": ["contact.qq"]
  }
}
```

**参数说明**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | string | 是 | 人设卡名称（1-200 个字符） |
| description | string | 是 | 人设卡描述（至少 10 个字符） |
| copyright_owner | string | 否 | 版权所有者（默认为上传者用户名） |
| tags | string | 否 | 标签（多个标签用逗号分隔） |
| is_public | boolean | 否 | 是否公开（默认为 false） |
| configs | array | 是 | 配置项列表 |
| sensitive_confirmation | object | 否 | 敏感信息确认（如果配置中包含敏感信息则必填） |

**响应示例**：
```json
{
  "code": 2000,
  "msg": "创建成功",
  "data": {
    "id": "uuid",
    "name": "我的人设卡",
    "description": "这是一个测试人设卡",
    "is_public": false,
    "is_pending": false
  }
}
```

**错误响应**：
- `400`：数据验证失败（名称长度不符、描述过短等）
- `401`：用户未认证
- `403`：用户被封禁
- `429`：超过上传限额（每日 10 次）
- `500`：创建失败

### 3. 确认敏感信息

**端点**：`POST /api/content/persona/{id}/confirm-sensitive/`

**说明**：确认配置中的敏感信息（5-11 位连续数字）不涉及个人隐私。

**权限**：需要登录，仅上传者可操作

**请求参数**：
```json
{
  "confirmation_text": "我已确认该文件在第 10 行的内容不涉及个人隐私信息",
  "sensitive_locations": ["contact.qq", "group.id"],
  "captcha_key": "abc123",
  "captcha_value": "42"
}
```

**参数说明**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| confirmation_text | string | 是 | 确认声明（必须包含"确认"和"隐私"关键词） |
| sensitive_locations | array | 是 | 敏感信息位置列表 |
| captcha_key | string | 是 | 验证码 key |
| captcha_value | string | 是 | 验证码答案 |

**验证码规则**：
- 使用数学验证码（加减乘除）
- 最多尝试 **10 次**
- 失败 10 次后进入 **1 分钟**冷却期

**响应示例**：
```json
{
  "code": 2000,
  "msg": "确认成功",
  "data": {
    "id": "uuid",
    "confirmation_text": "我已确认该文件在第 10 行的内容不涉及个人隐私信息",
    "ip_address": "192.168.1.1",
    "confirmed_at": "2024-01-01T12:00:00Z"
  }
}
```

**错误响应**：
- `400`：验证码错误或确认声明格式不正确
- `403`：只有创建者可以确认敏感信息
- `429`：验证码错误次数过多，进入冷却期

### 5. 更新配置项

**端点**：`PUT /api/content/persona/{id}/config/`

**说明**：更新人设卡的配置项，支持批量更新和配置块删除标记。

**权限**：需要登录，仅上传者可编辑，且人设卡必须为私有状态或未通过审核

**编辑限制**：
- 审核中的人设卡不能编辑
- 已通过审核的公开人设卡不能编辑
- 如需编辑，需先将人设卡转为私有

**请求参数**：
```json
{
  "updates": [
    {
      "section": "inner.meta.card",
      "key": "version",
      "value": "1.1",
      "comment": "更新版本号"
    }
  ],
  "deleted_sections": ["deprecated_section"]
}
```

**参数说明**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| updates | array | 否 | 要更新的配置项列表 |
| deleted_sections | array | 否 | 要标记为已删除的配置块名称列表 |

**响应示例**：
```json
{
  "code": 2000,
  "msg": "更新成功",
  "data": {
    "sections": [...]
  }
}
```

**错误响应**：
- `400`：请提供更新数据或配置项不存在
- `401`：请先登录
- `403`：无权限编辑此人设卡（审核中或已通过审核）
- `500`：更新失败

**错误响应**：
- `403`：只有上传者可以导出配置
- `500`：导出失败

### 7. 下载人设卡

**端点**：`GET /api/content/persona/{id}/download/`

**说明**：下载人设卡的 bot_config.toml 配置文件。

**权限**：需要登录，仅公开且已通过审核的人设卡可下载

**功能**：
- 导出配置项为 TOML 格式
- 增加下载计数
- 记录下载历史（用户 ID、人设卡 ID、下载时间）
- 返回文件响应（Content-Type: application/toml）

**响应**：
- 成功：返回 TOML 文件（Content-Disposition: attachment; filename="bot_config.toml"）
- 失败：返回 JSON 错误响应

**错误响应**：
- `401`：请先登录
- `403`：该人设卡未公开或正在审核中，无法下载
- `500`：下载失败

---

### 8. 切换公开/私有状态

**端点**：`POST /api/content/persona/{id}/toggle-public/`

**说明**：切换人设卡的公开/私有状态。

**权限**：需要登录，仅上传者可操作

**状态转换**：
- **公开转私有**：设置 `is_public=False`，保持 `is_pending=False`
- **私有转公开**：设置 `is_public=True`，`is_pending=True`，触发 AI 自动审核

**响应示例**：
```json
{
  "code": 2000,
  "msg": "已将人设卡设为公开，正在审核中",
  "data": {
    "id": "uuid",
    "is_public": true,
    "is_pending": true
  }
}
```

**错误响应**：
- `403`：只有上传者可以切换人设卡状态
- `404`：人设卡不存在
- `500`：操作失败

## 使用流程

### 上传私有人设卡

1. **解析 TOML 文件**
   - 调用 `POST /api/content/persona/parse-toml/` 上传文件
   - 检查解析结果和敏感信息

2. **确认敏感信息**（如果有）
   - 如果检测到敏感信息，需要确认
   - 获取验证码
   - 调用 `POST /api/content/persona/{id}/confirm-sensitive/` 确认

3. **创建人设卡**
   - 调用 `POST /api/content/persona/create-with-config/`
   - 设置 `is_public=false`
   - 提供配置项和敏感信息确认（如果有）

4. **编辑配置**（可选）
   - 调用 `GET /api/content/persona/{id}/config/` 获取配置
   - 调用 `PUT /api/content/persona/{id}/config/` 更新配置

### 上传公开人设卡

1. **按照私有人设卡流程创建**
   - 先创建为私有（`is_public=false`）

2. **转为公开并提交审核**
   - 调用 `POST /api/content/persona/{id}/toggle-public/`
   - 系统自动触发 AI 审核

3. **等待审核结果**
   - AI 审核通过：自动批准，`is_pending=False`
   - AI 审核拒绝：移交人工审核
   - AI 审核不确定：自动批准

4. **接收审核通知**
   - 系统发送站内信
   - WebSocket 实时推送（如果在线）

### 下载人设卡

1. **浏览公开人设卡列表**
   - 调用 `GET /api/content/persona/`
   - 筛选已通过审核的人设卡（`is_public=true`, `is_pending=false`）

2. **下载配置文件**
   - 调用 `GET /api/content/persona/{id}/download/`
   - 保存为 `bot_config.toml`

## 敏感信息检测

### 检测规则

系统自动检测配置中的 **5-11 位连续数字**，识别为可能的敏感信息（如 QQ 号、群号）。

**检测范围**：
- 所有配置项的值
- 字符串类型的值
- 数组和对象中的字符串值

**示例**：
- `12345`（5 位）：检测为敏感信息
- `12345678`（8 位）：检测为敏感信息
- `12345678901`（11 位）：检测为敏感信息
- `1234`（4 位）：不检测
- `123456789012`（12 位）：不检测

### 确认流程

1. **系统检测到敏感信息**
   - 在解析 TOML 文件时自动检测
   - 返回敏感信息列表

2. **用户确认**
   - 查看敏感信息位置
   - 输入确认声明（必须包含"确认"和"隐私"关键词）
   - 输入验证码

3. **系统记录**
   - 记录确认声明
   - 记录用户 IP 地址
   - 记录确认时间

### 确认声明模板

```
我已确认该文件在第 X 行的内容不涉及个人隐私信息
```

其中 X 为所有敏感信息所在的行号（如果有多个，用逗号分隔）。

## 审核流程

### AI 自动审核

当用户将人设卡设为公开时，系统自动触发 AI 审核：

1. **创建审核任务**
   - 使用 Celery 异步执行
   - 调用现有的 AI 审核服务

2. **审核决策**
   - **通过**：自动批准，`is_pending=False`
   - **不确定（unknown）**：自动批准，`is_pending=False`
   - **拒绝**：保持 `is_pending=True`，移交人工审核队列

3. **创建审核报告**
   - 记录审核决策
   - 记录审核原因
   - 记录违规类型（如果拒绝）

### 审核结果通知

审核完成后，系统自动发送通知：

1. **站内信**
   - 包含人设卡名称
   - 包含审核决策
   - 包含审核报告摘要
   - 如果拒绝，包含拒绝原因和违规类型

2. **WebSocket 推送**
   - 实时推送给在线用户
   - 包含审核结果

3. **查看审核报告**
   - 在人设卡列表中显示审核状态
   - 可以查看完整的审核报告

## 常见问题

### 1. 为什么我的上传被拒绝？

**可能原因**：
- 文件名不是 `bot_config.toml`
- 文件大小不在 1KB - 2MB 范围内
- 文件编码不是 UTF-8
- TOML 格式错误
- 缺少 `version` 字段
- 超过每日上传限额（10 次）

**解决方法**：
- 检查文件名和大小
- 使用 UTF-8 编码保存文件
- 验证 TOML 语法
- 确保包含 `version` 字段
- 等待次日重置限额

### 2. 为什么我不能编辑人设卡？

**可能原因**：
- 人设卡正在审核中（`is_pending=true`）
- 人设卡已通过审核且为公开状态（`is_public=true`, `is_pending=false`）

**解决方法**：
- 等待审核完成
- 将人设卡转为私有后再编辑

### 3. 如何处理敏感信息？

**建议**：
- 如果配置中包含真实的 QQ 号或群号，建议使用占位符（如 `00000`）
- 如果数字不是敏感信息（如年份、日期），可以正常确认
- 确认时需要输入验证码，防止自动化攻击

### 4. 验证码错误次数过多怎么办？

**规则**：
- 最多尝试 10 次
- 失败 10 次后进入 1 分钟冷却期

**解决方法**：
- 等待 1 分钟后重试
- 仔细查看验证码图片
- 确保输入正确的答案

### 5. 如何查看我的上传限额？

**查看方法**：
- 创建人设卡时，如果超过限额会返回错误消息
- 错误消息中包含剩余次数和重置时间

**限额规则**：
- 每个用户每天最多上传 10 个人设卡
- 限额在 UTC 零点自动重置

### 6. 下载的 TOML 文件包含被删除的配置块吗？

**规则**：
- 下载时不包含被删除的配置块
- 但会在被删除的位置创建空块并添加注释："# 此配置块已被作者删除"

**提醒**：
- 如果配置中存在被删除的配置块，下载前会弹出提醒对话框
- 对话框以表格形式列出所有被删除的配置块名称

## 技术细节

### TOML 解析

- 使用 Python `toml` 库解析文件
- 提取配置块、键值对、数据类型和注释
- 验证必需字段（`version`）
- 识别数据类型：string、integer、float、boolean、array、object

### TOML 导出

- 将 PersonaCardConfig 对象集合格式化为 TOML 文件
- 保留配置项注释
- 为被删除的配置块创建空块并添加注释
- 支持往返属性（解析 → 导出 → 解析 = 等价）

### 配置项存储

- 每个配置项创建一个 PersonaCardConfig 模型实例
- 复杂类型（array、object）序列化为 JSON 字符串存储
- 记录创建时间和更新时间
- 支持软删除（`is_deleted` 字段）

### 上传限额管理

- 使用 Redis 缓存计数
- 键名：`persona_upload_quota:{user_id}`
- TTL：到下一个 UTC 零点的秒数
- 自动重置

### 验证码验证

- 使用 django-simple-captcha
- 数学验证码（加减乘除）
- 重试次数存储在 Redis
- 冷却期存储在 Redis

## 安全考虑

1. **文件上传安全**
   - 验证文件名、大小、类型、编码
   - 使用唯一文件名存储
   - 存储在安全目录

2. **XSS 防护**
   - 对所有用户输入进行 HTML 转义
   - 使用 Django 的自动转义机制

3. **CSRF 保护**
   - 使用 Django 的 CSRF 中间件
   - 所有 POST/PUT/DELETE 请求需要 CSRF token

4. **权限控制**
   - 所有 API 端点需要用户认证
   - 验证用户是否有权限操作资源

5. **敏感信息保护**
   - 检测配置中的敏感信息
   - 要求用户确认
   - 记录确认声明和 IP 地址
   - 使用验证码防止自动化攻击

## 性能优化

1. **数据库优化**
   - 为常用查询字段添加索引
   - 使用 select_related 和 prefetch_related
   - 使用 bulk_create 批量创建配置项

2. **缓存策略**
   - 使用 Redis 缓存上传限额
   - 缓存验证码重试次数

3. **文件处理优化**
   - 流式读取大文件
   - 异步处理文件解析

4. **前端优化**
   - 使用虚拟滚动展示大量配置项
   - 懒加载配置块
   - 防抖处理敏感信息检测
