# 人设卡上传菜单配置说明

## 概述

本文档说明如何配置人设卡上传功能的菜单和路由。该功能在"内容管理"菜单下添加了两个新的子菜单项。

## 菜单结构

### 新增菜单项

在"内容管理"菜单下新增了以下两个菜单项：

1. **上传人设卡**
   - 路径：`/content/persona/upload`
   - 组件：`content/persona/upload/PersonaCardUploadWizard`
   - 图标：`ele-Upload`
   - 功能：提供分步上传向导，用户可以上传和配置人设卡

2. **我的人设卡**
   - 路径：`/content/persona/manage`
   - 组件：`content/persona/manage/PersonaCardList`
   - 图标：`ele-FolderOpened`
   - 功能：显示用户上传的所有人设卡列表，支持编辑、删除、下载等操作

### 完整菜单层级

```
内容管理 (/content)
├── 知识库管理 (/content/knowledge-base)
├── 人设卡管理 (/content/persona-card)
├── 上传人设卡 (/content/persona/upload)      [新增]
└── 我的人设卡 (/content/persona/manage)      [新增]
```

## 权限配置

### 角色权限

所有角色都可以访问新增的菜单项：

- **superadmin（超级管理员）**：完全访问权限
- **admin（管理员）**：完全访问权限
- **reviewer（审核员）**：完全访问权限
- **user（普通用户）**：可以上传和管理自己的人设卡

### 菜单按钮权限

#### 上传人设卡菜单按钮

- `persona_upload:ParseToml` - 解析TOML文件
- `persona_upload:CreateWithConfig` - 创建人设卡及配置
- `persona_upload:ConfirmSensitive` - 确认敏感信息

#### 我的人设卡菜单按钮

- `persona_manage:MyList` - 查询我的人设卡列表
- `persona_manage:GetConfig` - 获取配置项
- `persona_manage:UpdateConfig` - 更新配置项
- `persona_manage:ExportToml` - 导出TOML文件
- `persona_manage:Download` - 下载人设卡
- `persona_manage:TogglePublic` - 切换公开/私有状态
- `persona_manage:Delete` - 删除人设卡

## 应用配置

### 方法一：重新初始化菜单（推荐用于开发环境）

如果是开发环境且可以重置数据，执行以下命令：

```bash
# 激活 Conda 环境
conda activate mai_notebook

# 进入后端目录
cd backend

# 重新初始化菜单数据
python mainotebook/system/fixtures/initialize.py
```

### 方法二：手动添加菜单（推荐用于生产环境）

如果是生产环境，需要通过管理后台手动添加菜单：

1. 登录管理后台
2. 进入"系统管理" -> "菜单管理"
3. 找到"内容管理"菜单
4. 添加"上传人设卡"子菜单：
   - 菜单名称：上传人设卡
   - 路由地址：/content/persona/upload
   - 组件地址：content/persona/upload/PersonaCardUploadWizard
   - 组件名称：personaCardUpload
   - 图标：ele-Upload
   - 排序：3
   - 是否缓存：否
   - 侧边栏显示：是

5. 添加"我的人设卡"子菜单：
   - 菜单名称：我的人设卡
   - 路由地址：/content/persona/manage
   - 组件地址：content/persona/manage/PersonaCardList
   - 组件名称：personaCardManage
   - 图标：ele-FolderOpened
   - 排序：4
   - 是否缓存：是
   - 侧边栏显示：是

6. 为每个菜单添加相应的按钮权限（参考上面的"菜单按钮权限"部分）

7. 在"角色管理"中为相应角色授权新菜单

### 方法三：使用数据库脚本（推荐用于批量部署）

创建数据库迁移脚本来添加菜单：

```python
# backend/mainotebook/system/migrations/0004_add_persona_upload_menus.py
from django.db import migrations

def add_persona_upload_menus(apps, schema_editor):
    Menu = apps.get_model('system', 'Menu')
    
    # 获取"内容管理"菜单
    content_menu = Menu.objects.get(name="内容管理", web_path="/content")
    
    # 创建"上传人设卡"菜单
    upload_menu = Menu.objects.create(
        parent=content_menu,
        name="上传人设卡",
        icon="ele-Upload",
        sort=3,
        is_link=False,
        is_catalog=False,
        web_path="/content/persona/upload",
        component="content/persona/upload/PersonaCardUploadWizard",
        component_name="personaCardUpload",
        status=True,
        cache=False,
        visible=True
    )
    
    # 创建"我的人设卡"菜单
    manage_menu = Menu.objects.create(
        parent=content_menu,
        name="我的人设卡",
        icon="ele-FolderOpened",
        sort=4,
        is_link=False,
        is_catalog=False,
        web_path="/content/persona/manage",
        component="content/persona/manage/PersonaCardList",
        component_name="personaCardManage",
        status=True,
        cache=True,
        visible=True
    )

def remove_persona_upload_menus(apps, schema_editor):
    Menu = apps.get_model('system', 'Menu')
    Menu.objects.filter(
        web_path__in=[
            "/content/persona/upload",
            "/content/persona/manage"
        ]
    ).delete()

class Migration(migrations.Migration):
    dependencies = [
        ('system', '0003_alter_usernotificationpreference_message_type'),
    ]

    operations = [
        migrations.RunPython(add_persona_upload_menus, remove_persona_upload_menus),
    ]
```

然后执行迁移：

```bash
python manage.py migrate
```

## 验证配置

### 1. 检查菜单是否显示

登录系统后，在左侧菜单栏中应该能看到：

```
内容管理
├── 知识库管理
├── 人设卡管理
├── 上传人设卡  ← 新增
└── 我的人设卡  ← 新增
```

### 2. 检查路由是否正常

访问以下URL应该能正常加载页面（前端组件实现后）：

- `/#/content/persona/upload` - 上传向导页面
- `/#/content/persona/manage` - 我的人设卡列表页面

### 3. 检查权限是否生效

- 普通用户应该能看到"上传人设卡"和"我的人设卡"菜单
- 点击菜单应该能正常跳转，不会出现权限错误

## 前端组件开发

菜单配置完成后，需要开发对应的前端组件：

### 需要创建的组件

1. **上传向导组件**
   - 路径：`admin_web/src/views/content/persona/upload/PersonaCardUploadWizard.vue`

2. **人设卡列表组件**

3. **配置编辑器组件**
   - 路径：`admin_web/src/views/content/persona/upload/ConfigEditor.vue`
   - 功能：在线编辑TOML配置

4. **其他辅助组件**
   - `BasicInfoForm.vue` - 基本信息表单
   - `ConfigSection.vue` - 配置块组件
   - `ConfigItem.vue` - 配置项组件
   - `SensitiveInfoDialog.vue` - 敏感信息确认对话框
   - `DeletedBlocksDialog.vue` - 被删除块提醒对话框

详细的前端组件开发说明请参考任务列表中的阶段6。

## 注意事项

1. **路由守卫**：所有新增路由都需要用户登录才能访问，这由系统的全局路由守卫自动处理

2. **组件命名**：组件名称（component_name）必须与实际Vue组件的name属性一致，用于路由缓存

3. **图标选择**：使用Element Plus图标库中的图标，格式为 `ele-IconName`

4. **排序规则**：sort值越小越靠前，建议预留间隔以便后续插入新菜单

5. **缓存策略**：
   - 上传向导页面不缓存（cache=false），每次进入都是新的上传流程
   - 我的人设卡列表页面缓存（cache=true），提升用户体验
